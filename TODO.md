## Что можно улучшить

Список открытых задач — если ищете, с чего начать контрибьютить, посмотрите сюда.

### Инфраструктура

- [ ] **cover.png**
    - [ ] **Генерация `cover.png` работает нестабильно.** Скрипт сейчас лучше запускать отдельно, а не в общем пайплайне. Нужно либо починить скрипт, либо переписать на более предсказуемое решение (например, генерация из шаблона по `title` лекции).
    - [ ] **Автоматический fallback для `cover.png`.** Сейчас файл обязателен — имеет смысл сделать дефолтную обложку на уровне курса или шаблона, чтобы лекция без своей обложки не ломалась.
    - Возможно и вовсе хорошо будет перенести cover.png в автогенерацию средствами html/css, не думаю что когда-либо там будут какие-то иные картинки, но был расчёт на то, что в картинке можно указать больше информации, либо поменять если что, по сути сейчас картинка дублирует title/description и в карточке лекции/курса и так всё есть...
- [ ] **Линтер для front matter.** Проверка обязательных полей (`title`, `date`, `description`, `tags`) до коммита.
- [ ] **CI-проверка сборки.** GitHub Action, прогоняющий `hugo --gc --minify` на каждом PR.
- [ ] **Сжатие изображений в пайплайне.** Автоматический прогон через `oxipng` / `cwebp` при сборке.

### Контент и оформление

- [ ] **Единое именование лекций внутри курса.** В части курсов смешаны схемы (`lecture_01`, где-то `multithreading-05-21`) — привести к одному стилю в рамках каждого курса.
- [ ] **Code style для конспектов.** Сформулировать соглашения: длина абзацев, использование callout'ов, оформление определений и теорем.
- [ ] **Глоссарий / индекс терминов.** Сквозной список определений по курсам со ссылками на лекции.

### Шаблоны

- [ ] **Шорткод для теорем и определений.** Сейчас оформление неоднородное — нужен единый блок с нумерацией.
