## Что можно улучшить

Список открытых задач — если ищете, с чего начать контрибьютить, посмотрите сюда.

### Инфраструктура

- [ ] **Линтер для front matter.** Проверка обязательных полей (`title`, `date`, `description`, `tags`) до коммита.
- [ ] **CI-проверка сборки.** GitHub Action, прогоняющий `hugo --gc --minify` на каждом PR.
- [ ] **Сжатие изображений в пайплайне.** Автоматический прогон через `oxipng` / `cwebp` при сборке.

### Контент и оформление

- [ ] **Code style для конспектов.** Сформулировать соглашения: длина абзацев, использование callout'ов, оформление определений и теорем.
- [ ] **Глоссарий / индекс терминов.** Сквозной список определений по курсам со ссылками на лекции.

### Шаблоны

- [ ] **Шорткод для теорем и определений.** Сейчас оформление неоднородное — нужен единый блок с нумерацией.
- [ ] **Тёмная тема** Тёмная тема местами работает странно из-за коллизий в теме. Где-то daisy-ui, где-то что-то другое... в общем беда вайбкодинга

Если хотите взяться за задачу — откройте issue или сразу draft PR, чтобы зафиксировать, что работа идёт.
