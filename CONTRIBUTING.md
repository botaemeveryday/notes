# Контрибьютинг

Этот документ описывает, как внести правки в репозиторий: структуру файлов, front matter, шорткоды и процесс отправки Pull Request. Перед началом работы рекомендуется ознакомиться с [README.md](README.md).

## Способы участия

### 1. Сообщить об ошибке

Если вы заметили проблему, но не готовы исправлять её самостоятельно — откройте [Issue](https://github.com/botaemeveryday/notes/issues/new?template=content-error.md). Подходящие поводы:

- Ошибка в тексте, формуле или коде.
- Битое изображение или проблема с рендерингом.
- Неточность в объяснении или фактическая ошибка.
- Предложение по улучшению структуры или подачи материала.

### 2. Исправить или дополнить существующий конспект

1. Найдите нужную лекцию в `content/posts/<курс>/<лекция>/index.md`.
2. Внесите правки, соблюдая правила оформления (см. ниже).
3. Проверьте результат локально (`hugo server -D`).
4. Отправьте Pull Request.

### 3. Добавить новую лекцию

```bash
mkdir -p content/posts/<курс>/<идентификатор-лекции>/images
touch content/posts/<курс>/<идентификатор-лекции>/index.md
```

Минимальный `index.md`:

```yaml
---
title: "Название лекции"
description: "Краткое описание содержания"
date: 2026-05-07
tags:
  - C++
  - 2 семестр
draft: false
---
```

### 4. Добавить новый курс

1. Создайте директорию `content/posts/<курс>/`.
2. Положите в неё `_index.md` с описанием курса (см. соседние курсы как образец).
3. Добавьте `cover.png` для секции.
4. Создавайте лекции по схеме выше.

## Локальная сборка

Требования: Hugo (extended), Node.js, npm, Git.

```bash
git clone https://github.com/botaemeveryday/notes.git
cd notes

npm install            # установка Tailwind и зависимостей
npm run css            # сборка стилей
hugo server -D         # http://localhost:1313
```

`npm run css` нужно перезапускать после добавления новых Tailwind-классов, которых ещё не было в проекте — иначе классы не попадут в итоговый CSS.

## Правила оформления

### Именование

| Объект | Соглашение | Примеры |
|---|---|---|
| Папка курса | `kebab-case` | `database-design`, `operation-systems` |
| Папка лекции | `kebab-case`, по содержанию или номеру | `lecture_01`, `multithreading-05-21`, `template-2-04-23` |
| Изображения | произвольное имя, желательно осмысленное | `cover.png`, `process-tree.png` |

В рамках одного курса придерживайтесь единого стиля именования лекций — либо нумерованного (`lecture_01`, `lecture_02`), либо тематического (`allocator-02-26`, `casts-04-02`). Не смешивайте.

### Структура папки лекции

```
content/posts/<курс>/<лекция>/
├── index.md          # текст лекции
└── images/
    ├── cover.png     # обложка (обязательно)
    └── ...           # остальные изображения
```

### Front matter

Обязательные поля:

```yaml
---
title: "Название лекции"
description: "Короткое описание (1–2 предложения)"
date: 2026-05-07T12:00:00+03:00
tags:
  - Название курса
draft: false
---
```

`draft: true` скрывает страницу из публикации — используется для незавершённых материалов.

### Изображения

- Поддерживаемые форматы: PNG, JPG, SVG, WebP.
- Хранятся в `images/` внутри папки лекции.
- Рекомендуется сжимать растровые изображения (например, через [squoosh.app](https://squoosh.app) или `oxipng`).
- Файл `cover.png` обязателен для каждой лекции.

### Формулы и код

- Математика — через MathJax, синтаксис LaTeX: `$inline$` и `$$display$$`.
- Блоки кода — с указанием языка для подсветки: ` ```cpp `, ` ```python ` и т. д.
- Mermaid-диаграммы — через стандартный блок ` ```mermaid `.

### Шорткоды

В проекте определены готовые шорткоды (`layouts/shortcodes/`), используйте их вместо ручной HTML-разметки:

| Шорткод | Назначение |
|---|---|
| `callout` | Выделенный блок (note, warning, tip) |
| `card`, `cards` | Карточки для навигации |
| `compare` | Сравнение двух вариантов бок о бок |
| `spoiler` | Скрытый блок с решением/ответом |
| `key` | Отображение клавиатурной комбинации |
| `stat`, `stat-row` | Числовая статистика |
| `youtube` | Встраивание видео |
| `anki-download` | Ссылка на колоду Anki |

## Pull Request

1. Сделайте форк репозитория.
2. Создайте ветку с осмысленным именем: `git checkout -b fix/cpp-exceptions`.
3. Внесите изменения и проверьте их локально.
4. Откройте Pull Request в `main`.

### Формат коммитов и заголовка PR

Проект придерживается [Conventional Commits](https://www.conventionalcommits.org/ru/v1.0.0/). Заголовок имеет вид:

```
<type>(<scope>): <описание>
```

**Типы (`type`):**

| Тип | Когда использовать |
|---|---|
| `feat` | Новая лекция, курс или функциональность сайта |
| `fix` | Исправление ошибки, опечатки, битой ссылки |
| `docs` | Изменения в README, CONTRIBUTING и прочей документации |
| `style` | Правки оформления, не меняющие смысл (отступы, форматирование) |
| `refactor` | Реорганизация контента или кода без изменения смысла |
| `chore` | Зависимости, конфиги, CI, скрипты |

**Scope** — короткое указание на область изменения: имя курса (`cpp`, `databases`, `os`) или компонент сайта (`layouts`, `shortcodes`, `ci`). Можно опустить, если правка глобальная.

**Примеры:**

```
feat(cpp): лекция про concepts
fix(os): опечатка в lecture_03
docs: уточнить требования к cover.png
refactor(databases): единое именование лекций
chore: обновить hugo до 0.130
```

В описании PR коротко укажите, что изменилось и зачем. Если правка затрагивает несколько лекций — перечислите их.
